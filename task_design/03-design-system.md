# Фаза 3 — Дизайн-система ⬜

**Цель:** определить палитру, типографику, сетку и брейкпоинты — до того как трогаем компоненты. Все последующие фазы ссылаются на переменные отсюда.

**Зависит от:** [00-decisions.md](00-decisions.md) (реш. 1 — акцентный цвет).

## Шаги

### Палитра

- [ ] **3.1.** Пересмотреть `:root` в [main.css](../assets/css/main.css):
  - Нейтральные: `--surface-0` (фон), `--surface-1` (карточки), `--surface-2` (подвал/шапка), `--border`.
  - Текст: `--text-primary`, `--text-secondary`, `--text-muted`.
  - Бренд: `--brand`, `--brand-dark`, `--brand-contrast`.
  - Акцент (CTA): `--accent`, `--accent-dark`.
  - Состояния: `--success`, `--warning`, `--danger`.
- [ ] **3.2.** Проверить контраст AA/AAA для каждой пары текст/фон (сервис: [WebAIM](https://webaim.org/resources/contrastchecker/)). Критично для `--text-muted` на `--surface-0`.
- [ ] **3.3.** Опционально: заготовка под `prefers-color-scheme: dark` — те же переменные, переопределённые в `@media`.

### Типографика

- [ ] **3.4.** Решить по шрифтам. Текущая пара Cuprum + Playfair Display — спорная. Варианты:
  - (a) оставить, но подобрать веса и межстрочные;
  - (b) Body → Inter / Manrope / PT Sans; Heading → оставить Playfair для H1/H2, остальное body-шрифтом;
  - (c) один шрифт на всё (PT Sans / Inter).
- [ ] **3.5.** Определить модульную шкалу: `--fs-100` … `--fs-900` с коэффициентом 1.125 или 1.2 (major second / minor third).
- [ ] **3.6.** Для мобильной версии текущие `h1: 1.6rem` и т.д. перенести на переменные, не захардкоденные значения.
- [ ] **3.7.** `font-display: swap` в `@font-face` — чтобы текст не оставался невидимым пока грузятся шрифты.

### Spacing

- [ ] **3.8.** Текущая шкала `--space-xs…xxl` — оставить, но добавить `--space-3xl` (96px) для больших секционных отступов.
- [ ] **3.9.** Унифицировать вертикальный ритм: отступы между секциями = `--space-xxl` на десктопе, `--space-xl` на мобильном.

### Сетка и контейнеры

- [ ] **3.10.** Добавить утилитарные ширины: `--container-narrow: 720px` (для текстовых страниц), `--container: 1200px` (дефолт), `--container-wide: 1440px` (для лендинга/hero).
- [ ] **3.11.** Класс `.container--narrow`, `.container--wide`.

### Breakpoints

- [ ] **3.12.** Сейчас один `@media (max-width: 768px)`. Расширить:
  - `--bp-sm: 640px` — смартфон в альбомной / маленький планшет;
  - `--bp-md: 960px` — планшет;
  - `--bp-lg: 1200px` — десктоп;
  - `--bp-xl: 1440px` — большой экран.
- [ ] **3.13.** Использовать `min-width`-запросы (mobile-first) вместо текущих `max-width`.

### Элементы

- [ ] **3.14.** Определить стили кнопок: `.btn`, `.btn--primary`, `.btn--ghost`, `.btn--link`. Радиус, паддинги, состояния `:hover`, `:focus-visible`, `:active`, `:disabled`.
- [ ] **3.15.** Состояния фокуса (`:focus-visible`) — обязательно для a11y, сейчас используется браузерный дефолт.
- [ ] **3.16.** Утилитарные классы: `.stack-s/m/l` (вертикальный spacing), `.cluster` (горизонтальный flex с gap), `.grid-auto` (repeat auto-fit).

## Критерий готовности

- В `main.css` единый блок переменных, на который ссылаются все компоненты.
- Все текущие страницы визуально не сломаны после замены (регрессия проверяется клик по клику).
- Lighthouse Accessibility ≥ 95 (влияет контраст и focus).

## Риск

Смена базового шрифта меняет метрики (строки переносятся иначе). После замены — пройти по всем content-страницам и проверить, не «едут» ли заголовки и списки.
