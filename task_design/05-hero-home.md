# Фаза 5 — Hero на главной и сетка услуг ⬜

**Цель:** за первый экран главной продать ценность и дать 2 CTA. Текущая главная — три строки текста и список разделов в линию.

**Зависит от:** [03-design-system.md](03-design-system.md), [00-decisions.md](00-decisions.md) (реш. 1, 2, 7).

## Hero-блок

### Вариант структуры

```
[ H1: Юридическая помощь гражданам в <Городе> ]
[ Подзаголовок: стаж N лет · M выигранных дел · первичная консультация бесплатно ]
[ CTA-primary: Записаться ]   [ CTA-secondary: ☎ Позвонить ]
[ Trust-бейджи: диплом / палата / рейтинг ]

              [ Фото юриста / иллюстрация ]
```

### Шаги

- [ ] **5.1.** Создать отдельный layout для главной или дополнить [home.html](../layouts/home.html) — hero как partial `hero-home.html`.
- [ ] **5.2.** H1 и подзаголовок — в `site.Params` или в front-matter `content/_index.md` (новые поля: `hero_title`, `hero_subtitle`, `hero_cta`).
- [ ] **5.3.** Две CTA-кнопки: первая — `#contact-form` (якорь/модалка), вторая — `tel:`. На мобильном — стекируются вертикально.
- [ ] **5.4.** Trust-бейджи (3–4 штуки): иконка + короткий текст. SVG-спрайт `static/icons/sprite.svg`.
- [ ] **5.5.** Фото юриста справа — `<picture>` с WebP/AVIF и `<source>` по брейкпоинтам. На мобильном фото уходит под текст либо прячется.
- [ ] **5.6.** Если фото нет (реш. 2) — декоративная SVG-иллюстрация или геометрический блок с инициалами.

## Сетка услуг (замена текущих section-preview)

### Текущее состояние

[section-home.html](../layouts/_partials/section-home.html) выводит заголовок-ссылку, описание, summary и список из 5 дочерних страниц. Визуально — просто список, без карточек, без иконок, без hover.

### Целевое

Карточка (ссылка на весь контейнер):
```
[ иконка SVG ]
H3: Название раздела
p: описание 1–2 строки
→ Подробнее
```

- [ ] **5.7.** Создать partial `service-card.html` на замену (или вместо) section-home.
- [ ] **5.8.** Иконки — SVG inline из общего спрайта. Для 8 разделов нужно 8 иконок (можно использовать tabler-icons или нарисовать самим).
- [ ] **5.9.** Grid: `grid-template-columns: repeat(auto-fit, minmax(280px, 1fr)); gap: var(--space-l)`. На десктопе получится 3–4 в ряд, на планшете 2, на мобильном 1.
- [ ] **5.10.** Hover: чуть приподнять карточку (`transform: translateY(-2px)`), тень, цвет иконки меняется на акцент. Учесть `prefers-reduced-motion` из [10-performance-a11y.md](10-performance-a11y.md).
- [ ] **5.11.** Вся карточка — `<a>` (внутри heading — не ссылка, чтобы не было вложенного `<a>`). Либо использовать «link overlay» паттерн через `::before`.
- [ ] **5.12.** Убрать из карточки список из 5 дочерних страниц — это уменьшает визуальный вес. Если хотим сохранить — делать это на hub-странице `/napravleniya.html`, не на главной.

## Блоки под hero

Последовательность главной после hero:

1. «Что я делаю» (3 карточки: Консультации / Суд / Документы).
2. «Направления практики» (grid 8 карточек по разделам).
3. «Как я работаю» (3–4 шага: заявка → оценка → договор → работа).
4. Кейсы (2–3 превью, см. [06-trust-social-proof.md](06-trust-social-proof.md)).
5. Отзывы (карусель или grid 3 шт.).
6. «Обо мне» (короткий блок с фото и CTA на `/o-yuriste`).
7. FAQ (5–7 вопросов, со Schema.org FAQPage из [02-seo-schema.md](02-seo-schema.md)).
8. Финальный CTA-блок перед футером.

- [ ] **5.13.** Каждый блок — отдельный partial в `layouts/_partials/home/*.html`.
- [ ] **5.14.** Порядок и включение — через массив в front-matter `content/_index.md` (`sections = ["services", "directions", "process", "cases", "reviews", "about", "faq", "cta"]`) либо хардкодом в home.html.

## Критерий готовности

- На первом экране виден H1, подзаголовок, две CTA, trust-бейджи.
- Сетка услуг адаптивная: 3–4 / 2 / 1 столбца по брейкпоинтам.
- Карточки кликабельны всей площадью, с фокус-стилем.

## Риск

Блоков на главной становится много (8). Каждый требует контента. Если контента нет — выносим блок за флаг и не показываем (MVP: hero + services + process + CTA).
