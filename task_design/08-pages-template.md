## `page.html` — страница услуги

### Текущая структура

```
breadcrumbs
h1
time (дата)
content
tags
```

### Целевая структура

```
breadcrumbs
[интро-блок: H1 + лид-параграф + CTA-кнопки]
[grid: слева — содержание (TOC), справа — контент]
[блок «Как я помогу» — 3–4 пункта]
[блок «Кейсы по этой теме» — 1–2 карточки]
[блок «Что взять на консультацию» — чеклист]
[FAQ по теме]
[CTA-блок перед футером]
[related — другие услуги в этом разделе]
tags (если есть)
```

## Критерий готовности

- Страница услуги читается как статья (контент в узкой колонке), но с правильными CTA и блоками доверия.
- TOC работает на длинных страницах, прячется на коротких.
- Related-блок выводит релевантные страницы того же раздела.

## Риск

Шаблон становится сложнее — много front-matter полей. Если юрист не будет заполнять — все опциональные блоки должны быть с `{{ with }}` и не ломать разметку при отсутствии данных.
