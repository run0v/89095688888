# План редизайна и доработки

Источник — аудит существующего сайта от 2026-04-17 (CSS, шаблоны, контент).
Цель — превратить рабочий MVP в конверсионный сайт частнопрактикующего юриста.

## Принципы

- Минимум сторонних зависимостей. Hugo + ванильный CSS/JS, никаких бандлеров.
- Мобильный приоритет. На телефоне сайт должен звонить, писать в мессенджер, отправлять заявку за 3 тапа.
- Доверие > дизайн. Фото, стаж, кейсы, отзывы важнее анимаций.
- Ничего не ломать. Текущий контент и URL сохраняются; редизайн идёт слоями.

## Что осталось по 🟡 фазам

- **00:** заглушки проставлены Claude'ом, юрист валидирует и уточняет реальные данные (email, адрес, цены, часы).
- **01:** положить реальные файлы `favicon.svg`, `apple-touch-icon.png`, `site.webmanifest`, `og-default.jpg` в `static/`.
- **02:** добавить `Article` schema на страницы услуг (можно в `page.html`).
- **05:** заменить placeholder-инициал «Ю» в hero на реальное фото юриста.
- **06:** content-типы `otzyvy/` и `keysy/` + страница `/o-yuriste.html` — после появления контента.
- **07:** форма с UI и JS-валидацией, но без бэкенда. Telegram-бот через Cloudflare Worker — отдельная задача.
- **10:** subsetting шрифтов (убрать неиспользуемые Oswald, Russo One, PlayfairSC).
- **11:** полная мультигородская структура отложена до появления реальных кейсов и отзывов по СПб и Москве. Пока упоминание трёх городов — в hero-бейджах и Schema.org `areaServed`.

## Принцип ведения

- Галочку ставим после фактической проверки: `make hugo_serve` + клик + Lighthouse для фаз 1, 2, 10.
- Если по ходу меняется решение — фиксируем в [00-decisions.md](00-decisions.md), а не в теле фазы.
- Файлы фаз — это план, не журнал. Короткие, проверяемые шаги.
- Порядок не строгий: 1–3 можно делать параллельно, 4–9 зависят от 3 (дизайн-система).
