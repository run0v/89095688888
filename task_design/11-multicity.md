# Фаза 11 — Мультигород: Петрозаводск, СПб, Москва ⬜

**Цель:** дать сайту локальное присутствие в трёх городах без дорвейного контента. Цель — попасть в локальную выдачу Яндекса/Google по запросам «юрист в Петрозаводске», «юрист СПб», «юрист Москва» + смежным.

**Зависит от:** [00-decisions.md:13](00-decisions.md) (список городов), [02-seo-schema.md](02-seo-schema.md), [05-hero-home.md](05-hero-home.md), [08-pages-template.md](08-pages-template.md).

## Архитектура

### URL-структура

```
/                             → Петрозаводск (home, без префикса)
/napravleniya.html            → общий хаб направлений (Петрозаводск)
/nasledstvo/                  → раздел (Петрозаводск)
/nasledstvo/spory.html        → услуга (Петрозаводск)

/spb/                         → СПб home
/spb/napravleniya.html
/spb/nasledstvo/
/spb/nasledstvo/spory.html

/moskva/                      → Москва home
/moskva/napravleniya.html
/moskva/nasledstvo/
/moskva/nasledstvo/spory.html
```

Общие страницы (одна на весь сайт):
- `/o-yuriste.html` — один юрист, не дублировать.
- `/otzyvy.html`, `/keysy.html` — общие, но с фильтром по городу.
- `/privacy-policy.html`, `/terms-of-service.html`.
- `/zayavka.html`.

### Hugo-реализация

Через **content mount** с параметризацией не получится чисто — нужен дубль контента. Варианты:

**(a) Мультисайт через `config/<lang>`** — нет, Hugo мультисайтинг привязан к языкам, для городов это хак.

**(b) Отдельные секции** — `content/spb/` и `content/moskva/` с полным деревом внутри, `content/` (root) = Петрозаводск. Разделы дублируются, но front-matter тянется из одного источника.

**(c) Подстановка через shortcodes и Params** — одна структура `content/`, в шаблонах выводим город через `.Params.city` или из URL. **Не годится** — нужны разные URL.

**Выбираем (b).** Чуть больше файлов, но прозрачно для Hugo и для SEO.

- [ ] **11.1.** Создать `content/spb/` и `content/moskva/` с зеркалом структуры `content/` (разделы, `_index.md`).
- [ ] **11.2.** В `_index.md` каждой city-секции — front-matter:
  ```toml
  [params]
  city = "Санкт-Петербург"
  city_slug = "spb"
  city_prepositional = "Санкт-Петербурге"  # в СПб
  city_genitive = "Санкт-Петербурга"       # юрист СПб
  courts = ["Куйбышевский районный суд", "Невский районный суд", "Санкт-Петербургский городской суд", "Арбитражный суд города СПб и Ленинградской области"]
  price_multiplier = 1.15
  ```
- [ ] **11.3.** Каждая услуга внутри city-раздела — отдельный `.md` с локализованным текстом (минимум: лид-параграф + блок «как работаю в <городе>» + локальные суды).

### Определение города в шаблонах

- [ ] **11.4.** Helper-partial `_partials/helpers/city.html`, возвращающий city-объект по `.Page`:
  - если путь начинается с `/spb/` → city = СПб;
  - если `/moskva/` → Москва;
  - иначе → Петрозаводск (default).
- [ ] **11.5.** Во всех шаблонах, где выводится город (hero, NAP, breadcrumbs, Schema.org) — брать из этого helper'а.

## Контент-стратегия

### Обязательно уникальное на каждый город

- [ ] **11.6.** **Hero H1** — «Юрист в <городе>» (в падеже).
- [ ] **11.7.** **Лид-параграф** на главной — 2–3 абзаца про практику именно в этом городе. Упомянуть местные суды (по 2–3 штуки).
- [ ] **11.8.** **Кейсы** — минимум 2 уникальных из каждого города. Если нет реальных из СПб/Москвы — пока делаем только Петрозаводск и **не публикуем** city-версии до появления контента.
- [ ] **11.9.** **Отзывы** — минимум 2 местных на город. Без них city-страница выглядит подозрительно.
- [ ] **11.10.** **Страница услуги** — внутри текста заменяются упоминания судов и органов на локальные (через shortcode `{{< court city="spb" >}}` или через front-matter).
- [ ] **11.11.** **FAQ** — 1–2 вопроса специфичных для региона (например, для Карелии — особенности земельных споров в национальных районах; для СПб — коммунальные дела; для Москвы — реновация).

### Допустимо общее

- Структура сервиса, прайс (с мультипликатором из `price_multiplier`).
- Описание процесса работы («Как я помогу»).
- Юридические блоки, ссылки на нормы федерального уровня.
- «Обо мне» — один юрист на все города.

## Schema.org

- [ ] **11.12.** На главной (root, `/`) и на city-главных (`/spb/`, `/moskva/`) — отдельный `LegalService` JSON-LD с соответствующим `areaServed`.
  - root → `areaServed: [{"@type": "City", "name": "Петрозаводск"}, {"@type": "State", "name": "Республика Карелия"}]`
  - /spb/ → `areaServed: {"@type": "City", "name": "Санкт-Петербург"}`
  - /moskva/ → `areaServed: {"@type": "City", "name": "Москва"}`
- [ ] **11.13.** `sameAs` — ссылки на Яндекс.Бизнес, 2ГИС, профили мессенджеров. Если нет карточек — не врать, оставить пустым.

## Навигация

### Переключатель городов

- [ ] **11.14.** В шапке справа от меню — компактный селектор: `📍 Петрозаводск ▾` → dropdown с тремя городами.
- [ ] **11.15.** При клике на другой город — переход на соответствующую страницу этого города (логика сопоставления через одинаковые slug'и). Если соответствия нет — на главную города.
- [ ] **11.16.** Выбор города запоминаем в localStorage. При следующем визите с прямого захода на root — опционально redirect на последний выбранный (с toast'ом «вы смотрите СПб-версию, вернуться на Петрозаводск»).
- [ ] **11.17.** Не делать автоопределение по geo-IP — это раздражает и часто врёт (особенно с VPN и мобильным интернетом).

### Внутренняя перелинковка

## Проверка на дорвей

Тест перед публикацией: взять любую city-страницу услуги и **убрать упоминание города**. Если текст остаётся нечитаемым/неполным — хорошо, страница действительно локализована. Если читается так же — это дорвей, добавить локальный контент.

## Критерий готовности

- Все три города имеют полноценные главные с hero, услугами, минимум 2 кейсами и 2 отзывами.
- Переключатель городов работает, не теряет страницу при смене.
- Каждая city-страница canonical сама на себя.
- В Яндекс.Вебмастере нет ошибок «дубли страниц» между городами.
- Schema.org `LegalService` уникальна для каждого города с правильным `areaServed`.

## Риск

**Главный:** без реальных кейсов и отзывов из СПб и Москвы — эти города будут выглядеть фейково. Рекомендация: на старте публиковать только Петрозаводск + страницу «География: также принимаю заявки из СПб и Москвы онлайн». СПб и Москву — включать после того как появится минимум 2 реальных кейса на каждый город.

**Второстепенный:** тройной объём контента = тройная работа по поддержке. Держать city-специфичный контент в одном месте на страницу (front-matter + 1–2 абзаца), избегать текстов-дублей.

**Технический:** Hugo при большом количестве страниц × 3 города может замедлиться на билде. На текущем объёме (~30 страниц × 3 = 90) не критично, но следить.
