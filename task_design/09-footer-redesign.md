# Фаза 9 — Редизайн футера ⬜

**Цель:** превратить футер из технической сноски в последний конверсионный экран + навигационную карту.

**Зависит от:** [01-tech-head.md](01-tech-head.md), [03-design-system.md](03-design-system.md), [00-decisions.md](00-decisions.md) (реш. 4, 10, 11).

## Текущее состояние

[footer.html](../layouts/_partials/footer.html):

- Список всех разделов.
- Юридическое меню (политика, условия).
- Текст про cookies.
- Строка генерации sitemap + количество страниц.
- Copyright.

Проблемы: нет NAP, нет контактов, нет CTA, есть мусор (sitemap date + count).

## Целевая структура

```
┌──────────────────────────────────────────────────────────────┐
│  Логотип / имя юриста       Направления практики (2 колонки) │
│  Короткое описание                                           │
│  NAP: адрес, телефон,                                        │
│       email, мессенджеры    Услуги       │  О сайте          │
│                             — Консульт.  │  — О юристе       │
│  Часы работы                — Суд        │  — Контакты       │
│                             — Документы  │  — Отзывы         │
│                                          │  — Кейсы          │
│  ────────────────────────────────────────────────────────── │
│  © 2007–2026  ·  Политика конф.  ·  Условия  ·  Карта сайта │
└──────────────────────────────────────────────────────────────┘
```

## Шаги

### Контактная колонка

- [ ] **9.1.** Новый partial `footer-contacts.html`:
  - Название / имя юриста (H2 визуально, семантически `<p><strong>`).
  - 1–2 строки описания.
  - Телефон (кликабельный `tel:`) + иконка.
  - Email (кликабельный `mailto:`) + иконка.
  - Мессенджеры: WhatsApp, Telegram — иконки-ссылки.
  - Адрес (из реш. 10) — если есть.
  - Часы работы (из реш. 11).

### Навигационные колонки

- [ ] **9.2.** Разделить текущий `<nav class="footer-sections">` на две группы:
  - «Услуги» — три типа услуг (Консультации / Суд / Документы).
  - «Направления практики» — 8 разделов (текущий список).
  - «О сайте» — О юристе, Контакты, Отзывы, Кейсы.
- [ ] **9.3.** Сетка: `grid-template-columns: 1.5fr 1fr 1fr 1fr` на десктопе, 1 колонка на мобильном (accordion — опционально).

### Copyright-строка

- [ ] **9.4.** Убрать строку «Генерация sitemap: … [ N ]» — это служебная информация, не для прода (см. [01-tech-head.md](01-tech-head.md)).
- [ ] **9.5.** Формат: `© 2007–{{ now.Year }} · <имя юриста> · Политика конфиденциальности · Условия использования · Карта сайта`.
- [ ] **9.6.** Ссылка на `sitemap.xml` — технически есть, но в UI её прятать смысла нет; оставить как «Карта сайта» в copyright-строке.

### Текст про cookies

- [ ] **9.7.** Пересмотреть «Сайт не использует Cookie, не собирает персональные данные, и не обрабатывает последние». После:
  - добавления формы (собираются ПДн) — неправда;
  - карты Яндекса (реш. 10) — cookies появляются;
  - Яндекс.Метрики (если подключим) — cookies появляются.
- [ ] **9.8.** Заменить на корректную формулировку или удалить, переложив на [privacy-policy](../content/privacy-policy.md).

### CTA в футере (опционально)

- [ ] **9.9.** Мини-блок над сеткой: «Нужна юридическая помощь? Запишитесь на консультацию» + одна CTA-кнопка. Выводим только если пользователь дошёл до футера, не увидев форму — значит, ему нужен ещё один шанс.
- [ ] **9.10.** Решить по визуальному весу: либо это полноценный блок, либо одна строка. Тестировать.

### Схемы разметки

- [ ] **9.11.** Копия `LegalService` Schema.org уже в [02-seo-schema.md](02-seo-schema.md) — не дублировать, просто убедиться, что NAP в футере совпадает с NAP в JSON-LD (Google ругается на рассогласование).

## Критерий готовности

- В футере есть NAP, часы, мессенджеры, навигация по всем разделам.
- Текст про cookies соответствует действительности.
- Нет служебного мусора.
- Мобильная версия — читаемый столбец, не каша.

## Риск

Перегруженный футер на мобильном = бесконечный скролл. Если колонок много — на мобильном делать аккордеон (`<details>`) или скрывать менее важные колонки.
