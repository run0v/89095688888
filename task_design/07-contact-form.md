# Фаза 7 — Форма заявки и страница «Контакты» ⬜

**Цель:** дать каналу связи, альтернативному звонку. Для клиентов, которые не готовы сразу звонить, форма снижает порог входа.

**Зависит от:** [00-decisions.md](00-decisions.md) (реш. 4, 5, 6, 10), [03-design-system.md](03-design-system.md).

## Форма заявки

### Поля (минимум)

- `name` — имя (обязательное).
- `phone` — телефон (обязательное, маска).
- `message` — суть вопроса (необязательное).
- `consent` — чекбокс согласия на обработку ПДн (обязательный).
- `_honey` — honeypot-поле, скрытое CSS, заполненное = спам.

### Размещение

- Якорный блок `#zayavka` в конце главной.
- Модалка, открываемая по CTA-кнопкам «Записаться» из шапки и hero.
- Отдельная страница `/zayavka.html` — на случай прямых ссылок / QR-кодов.

### Шаги

- [ ] **7.1.** Partial `form-lead.html` — разметка формы + ARIA-атрибуты (`aria-describedby`, `aria-required`).
- [ ] **7.2.** Валидация HTML5 (`required`, `pattern`, `type=tel`) + JS-улучшение (маска телефона, inline-ошибки).
- [ ] **7.3.** Маска телефона — без сторонних либ, ~40 строк JS (`+7 (XXX) XXX-XX-XX`).
- [ ] **7.4.** Honeypot-поле + минимальное время заполнения формы (≥ 2 сек) как анти-спам.
- [ ] **7.5.** Модалка — `<dialog>` (нативный, с fallback на `role="dialog"` для старых браузеров), `Esc` + клик по оверлею = закрыть.
- [ ] **7.6.** После отправки: inline-сообщение «Заявка принята, перезвоним в течение N минут» + `aria-live="polite"`.

### Бэкенд (реш. 5)

Варианты из [00-decisions.md](00-decisions.md):

**(a) Telegram-бот** (рекомендую для MVP):
- [ ] **7.7a.** Создать бота через `@BotFather`, получить token.
- [ ] **7.8a.** Serverless-прокси (Cloudflare Workers / Yandex Cloud Function) — принимает POST с формы, шлёт в Telegram-чат юриста. Token — в env, не в клиентском коде.
- [ ] **7.9a.** CORS + rate-limit (10 заявок / IP / час).

**(b) Formspark / Getform** (быстрее всего):
- [ ] **7.7b.** Создать эндпоинт, `action=https://...` прямо в форме.
- [ ] **7.8b.** Настроить email-уведомления на почту юриста.

**(c) Свой email-шлюз:**
- [ ] **7.7c.** Использовать SMTP / Resend / Mailgun через serverless.

- [ ] **7.12.** Если планируется рассылка — отдельный чекбокс согласия на получение сообщений.

## Страница «Контакты»

### Содержание

- NAP: name, address, phone — те же данные, что в Schema.org.
- Часы работы — таблица или список.
- Способы связи: телефон, WhatsApp, Telegram, email.
- Карта — только если есть офис (реш. 10). Если карта через `<iframe>` Яндекс.Карт — учесть cookies (см. [09-footer-redesign.md](09-footer-redesign.md) и предупреждение в текущем футере).
- Форма заявки (та же, что на главной).

## Критерий готовности

- Тестовая заявка приходит в канал Telegram / на почту.
- Форма работает без JS (graceful degradation) хотя бы в mailto-варианте.
- Валидация показывает понятные ошибки на русском.
- Lighthouse Accessibility не падает из-за формы.

## Риск

Без rate-limit на бэкенде бот закидают спамом. Обязательно — honeypot + rate-limit + проверка referer.
