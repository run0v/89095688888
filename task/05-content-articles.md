## Шаги (единообразно для каждой статьи)

- [ ] **5.1.** Создать `content/<slug-раздела>/<slug-статьи>.md` (через `make newpost` или вручную).
- [ ] **5.2.** Front matter:
  ```toml
  +++
  title = '<Название статьи>'
  date = 2026-04-16
  draft = false
  description = '<meta description>'
  weight = <для порядка внутри раздела>
  +++
  ```
- [ ] **5.3.** Структура тела: «С чем приходят», «Что делаем», «Стоимость/сроки», «Результат/кейс», CTA.
- [ ] **5.4.** Проверить: статья показывается в списке на странице раздела (`section.html`) и в блоке «первые 5» на карточке главной (`section-home.html`).

## Критерий готовности

- Для каждого раздела есть минимум одна дочерняя статья (чтобы раздел не был пустым).
- Все статьи из черновика созданы и опубликованы (`draft = false`).
- Хлебные крошки на статье отображают путь: Главная → Раздел → Статья.

## Заметка

Фазу можно выполнять итеративно: не обязательно закрывать все статьи разом. Но пустой раздел (без дочерних страниц) выглядит плохо — минимум одну статью в каждый.
